多环境多配置(mode : production | development | none)
1、webpack读取config文件时，默认会将CLI配置的命令合并到config导出的对象中。因此可以在准备config对象时接收两个参数
module.exports=(env,arg)=>({
env : production|none|development
       arg : 命令行其他参数
})
这里就可以根据不同的env，设定config不同的mode

2、建议项目准备多项目配置文件
webpack-cli在启动时可以指定读取文件
$ webpacl --config webpack.config.pro.js
webpack.config.common.js  // 公共配置
webpack.config.pro.js.   // 指定生产环境的设置，例如source-map，mode
webpack.config.dev.js.   // 指定开发环境的配置
这里可以利用 webpack-merge 插件，将配置对象进行合并(不是覆盖)

3、常用插件
DefinePlugin
是用来为我们代码中注入全局成员的，多第三方模块都是通过这个成员去判断运行环境，从而决定是否执行例如打印日志之类的操作。打包后，会自动在打包后文件中直接替换。
webpack.config.js
plugins: [
    new webpack.DefinePlugin({
      API_BASE_URL: '"https://api.example.com"'
   // 注意这里需要传入一个字符串字面量，否则打包后会直接还出现bug
    })
  ]
// 业务代码
console.log(API_BASE_URL)
// bundle.js
console.log("https://api.example.com”)

Mini CSS Extract Plugin
css单独提出文件，不在打包到bundle.js，通过styele便签引入。这样可以减小打包js文件大小。
OptimizeCssAssetsWebpackPlugin
对单独提出的css文件进行压缩

TerserWebpackPlugin
js文件压缩

这里webpack提供了两种配置方式
plugin方式
插件方式可以只需要css文件进行压缩，js仍会自动压缩
plugin:[
new OptimizeCssAssetsWebpackPlugin()
]
optimization方式
配置到优化项时，如果只配置css的压缩会导致js不再压缩，这时就需要js的压缩了。官网推荐这种方式，因为压缩的配置就放在一个地方了，方便管理。
optimization: {
    minimizer: [
      new TerserWebpackPlugin(),
      new OptimizeCssAssetsWebpackPlugin()
    ]
  }
